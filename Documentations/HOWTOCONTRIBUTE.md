# HOW TO CONTRIBUTE

* Thank you for your interest in contributing to the AREA project !
You will find all information on how to contribute in this file.

---

## Getting Started
Before making any changes, please check existing issues to see if someone has already reported or is working on your idea. For small changes (typos, documentation), feel free to open a pull request directly and for larger changes, please open an issue first to discuss your proposal.

---

## How to contribute

### Report bugs
- Check if the bug as already been reported in the [issues](../../../issues)
- Create a new issue using this template:
```bash
Title : [BUG] <descriptive title>

Description :
# Describe the bug
A clear description of the bug

# How to reproduce
Clear steps to reproduce the bug

# Expected behaviour and actual behaviour
What was expected to happen and what actually happened

# Environment
Mobile or Web, Browser, Versions, etc.
```
<br>

### Suggest features
- Check the existing [issues](../../../issues) to see if the feature you want to included has been discussed.
- Create a new issue using this template:
```bash
Title : [FEATURE] <descriptive title>

Description :
# Describe the feature
A clear description of the feature you would like to see
```
<br>

### Code contribution
- Fork the repository
- Create a branch for your feature or bug fix following the [style guidelines](#style-guidelines)
- Make your change and push following the [style guidelines](#style-guidelines)
- Update documentation
- Update fork with latest changes from upstream
- Open a pull request from your fork to our main branch

---

## Style Guidelines

### Branch name
- Feature: `feature_<feature-name>`
- Bug fix: `fix_<bug-name>`
- Documentation: `doc_<doc-name>`

<br>

### Commit Message
```text
<TYPE>: <description>
```
Types:
- add a new feature : `ADD`
- change an existing feature : `FIX`, `UPDATE`
- delete something : `DEL`, `RM`
- merge something in main branch : `MERGE`

<br>

### Documentation
- Update README.md or any other file in the Documentations directory
